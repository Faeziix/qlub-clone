Autonomous Issue Loop

How /loop /next-issue drives the deterministic issue-cycle workflow — select → implement ↔ review → ship — one GitHub issue at a time, for Faeziix/qlub-clone.

deterministic JS control flow AI agent (fresh context each spawn) success / approved path stop / pause for human decision

2 · The inner convergence loop (one issue)

Why it can't spin forever and why the reviewer can't rubber-stamp.

sequenceDiagram
    autonumber
    participant JS as Workflow (JS)
    participant I as 🤖 Implementer
    participant R as 🤖 Independent Reviewer
    participant GH as GitHub / branch

    JS->>I: implement round 1 — issue + acceptance criteria
    I->>GH: commit + push to feat/issue-N
    JS->>R: review round 1 — diff vs main, run tests
    R-->>JS: approved = false, with blocking items
    Note over JS: rounds left, thread blocking items forward
    JS->>I: implement round 2 — plus reviewer blocking list
    I->>GH: commit fixes
    JS->>R: review round 2
    R-->>JS: approved = true
    Note over JS: break, go to Ship — ready PR
    JS->>GH: open PR, Closes #N

3 · Loop-control states

stateDiagram-v2
    [*] --> Selecting
    Selecting --> NoWork: no unblocked issue
    Selecting --> Working: claimed issue + branch
    Working --> Reviewing: implement round
    Reviewing --> Working: not approved, rounds left
    Reviewing --> Approved: review approved
    Working --> Exhausted: cap reached, not approved
    Approved --> ShipReady: open READY PR
    Exhausted --> ShipDraft: open DRAFT PR + pause
    ShipReady --> Selecting: continue loop, next issue
    ShipDraft --> [*]: stop for human
    NoWork --> [*]: stop, report blocked or HITL

What makes it trustworthy

  • bounded The while (round < MAX_ROUNDS) guarantees at most 3 implement↔review passes — no infinite ping-pong.
  • independent Each review is a fresh agent reading the diff and running tests itself, so it verifies rather than trusts the implementer's summary.
  • clean context Every box marked 🤖 is a brand-new subagent — the main loop only keeps short summaries, so context never bloats across issues.
  • fail-safe Non-convergence never ships silently: it opens a draft PR, comments, and pauses the whole loop for you.
  • dependency-aware Selection only picks issues whose every "Blocked by" is closed, so the graph order is respected and HITL gates (legal #5, Postgres #6) hard-stop the loop.
